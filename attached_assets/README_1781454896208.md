## What This Is

Infinite Us has three deployable layers and a shared package workspace:

|Layer        |Description                                                             |Deploy Target     |
|-------------|------------------------------------------------------------------------|------------------|
|`apps/web`   |React 19 + Vite SPA — the game, dashboard, and all relationship features|Vercel (free)     |
|`apps/api`   |Express 5 + Socket.io — real-time game rooms, REST endpoints            |Render (free tier)|
|`apps/mobile`|Expo 53 React Native — Phase 2                                          |EAS / Expo Go     |
|`packages/*` |Shared DB schema, types, validators, config                             |— (imported)      |

The web app is a **PWA** (installable, offline-capable). The API uses **persistent WebSocket** connections via Socket.io — it cannot run as serverless functions. It needs a persistent server (Render free tier works fine).

## Quick Start — New GitHub + Replit Account

Follow these steps **in order**. Do not skip steps.

### Step 1 — Create Your GitHub Repository

1. Go to [github.com](https://github.com) and create a new account (or sign in to your new account).
1. Click **New repository** (the `+` in the top-right corner).
1. Set the following:
- **Repository name:** `infinite-us`
- **Visibility:** Private (recommended until you’re ready to launch)
- **DO NOT** initialize with a README, .gitignore, or license — leave all boxes unchecked.
1. Click **Create repository**.
1. Copy the repository URL — it will look like `https://github.com/YOUR_USERNAME/infinite-us.git`.

### Step 2 — Create Your Replit Account and Connect It to GitHub

1. Go to [replit.com](https://replit.com) and create a new account.
1. After signing in, click **Create Repl** in the sidebar.
1. Choose **Import from GitHub**.
1. Authorize Replit to access your GitHub account when prompted.
1. Paste your `infinite-us` repository URL.
1. Replit will clone the (currently empty) repo and open it as a workspace.

> **Note:** If the repo is empty, Replit may show an error. That is expected. Continue to Step 3 — you will push the initial file structure from inside Replit.

### Step 3 — Initialize the Workspace in Replit

Open the **Shell** tab in Replit and run the following commands one at a time:

```bash
# Confirm you are in the correct directory
pwd
# Should print: /home/runner/infinite-us (or similar)

# Initialize git if not already done
git init
git remote add origin https://github.com/YOUR_USERNAME/infinite-us.git

# Create the root workspace files
# (you will add real code in subsequent steps)
touch README.md
touch .gitignore
touch package.json
touch pnpm-workspace.yaml
touch tsconfig.json
touch .npmrc
```

### Step 4 — Install pnpm (the Package Manager)

Replit uses Node.js with npm by default. Switch to pnpm:

```bash
npm install -g pnpm@9
pnpm --version
# Should print 9.x.x
```

> If pnpm is already available in your Replit environment, skip the install and just verify the version.

### Step 5 — Create the Folder Structure

Run this full scaffold command in the Replit Shell:

```bash
mkdir -p \
  apps/web/src/{features/{auth,cards,game,growth,mirror,bonds,archive,journey},components/{ui,layout},hooks,lib/{constants,utils},pages,providers,store} \
  apps/web/public \
  apps/api/src/{modules/{blend,rooms,bonds,journey,admin},middleware,lib,websocket} \
  apps/mobile/app/{components,tabs} \
  apps/mobile/{constants,context,data,hooks} \
  packages/db/src/schema \
  packages/db/drizzle \
  packages/types/src \
  packages/validators/src \
  packages/config \
  docs \
  scripts \
  .github/workflows
```

### Step 6 — Copy Root Configuration Files

Paste the following files into Replit using the file editor. Start with the root files documented in the [Folder Structure](#folder-structure) and [Architecture Decisions](#architecture-decisions) sections below.

### Step 7 — First Commit and Push

```bash
git add -A
git commit -m "chore: initialize workspace structure"
git branch -M main
git push -u origin main
```

Refresh your GitHub repository. You should now see all the folders.

### Step 8 — Set Environment Secrets in Replit

In Replit, click **Secrets** (the lock icon in the left sidebar) and add each variable listed in the [Environment Variables Reference](#environment-variables-reference) section below.

These secrets are injected automatically at runtime and are never committed to git.

## Project Architecture

```
infinite-us/
├── apps/
│   ├── web/                  # React 19 + Vite 7 — SPA frontend
│   ├── api/                  # Express 5 + Socket.io — backend
│   └── mobile/               # Expo 53 React Native — Phase 2
├── packages/
│   ├── db/                   # Drizzle ORM + PostgreSQL schema (shared)
│   ├── types/                # Shared TypeScript types (shared)
│   ├── validators/           # Shared Zod validation schemas (shared)
│   └── config/               # Shared tsconfig, eslint, prettier
├── docs/                     # Architecture and design documentation
├── scripts/                  # Seed scripts, codegen utilities
├── .github/
│   └── workflows/            # CI/CD GitHub Actions
├── package.json              # Root workspace config
├── pnpm-workspace.yaml       # pnpm monorepo definition
├── tsconfig.json             # Root TypeScript config
└── .npmrc                    # pnpm config (security settings)
```

### Request Flow

```
User Browser (PWA)
    │
    ▼
apps/web  (React SPA on Vercel)
    │  REST calls to /api/*
    │  WebSocket to /api/socket.io
    ▼
apps/api  (Express + Socket.io on Render)
    │  Drizzle ORM queries
    ▼
PostgreSQL  (Supabase or Neon — free tier)
```

## Environment Variables Reference

### `apps/api` — Server Environment

Create a `.env` file inside `apps/api/` (never commit this — it is in `.gitignore`).
Add each variable to Replit Secrets with the exact same key names.

```env
# ── Database ───────────────────────────────────────────────────────────────────
# PostgreSQL connection string from Supabase or Neon
# Format: postgresql://USER:PASSWORD@HOST:5432/DATABASE
DATABASE_URL=postgresql://...

# ── Authentication ─────────────────────────────────────────────────────────────
# Random string, minimum 32 characters. Used to sign JWTs with HMAC-SHA256.
# Generate with: node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
AUTH_SECRET=replace_with_64_char_random_hex_string

# ── Admin Panel ────────────────────────────────────────────────────────────────
# Passphrase for the hidden 7-tap admin panel (tap the title 7× in 3 seconds)
ADMIN_PASSPHRASE=replace_with_your_admin_passphrase

# ── CORS ───────────────────────────────────────────────────────────────────────
# Comma-separated list of allowed frontend origins
# In development: http://localhost:5173
# In production: https://your-app.vercel.app
CORS_ORIGIN=http://localhost:5173

# ── Server ─────────────────────────────────────────────────────────────────────
PORT=3000
NODE_ENV=development
```

## Architecture Decisions

### Why pnpm Workspaces?

The web app, API server, and mobile app share TypeScript types, Zod validators, and the database schema. Keeping them in a monorepo means one `pnpm install`, consistent versioning, and no copy-paste between packages. Each `apps/*` can still be deployed independently.

### Why Not Netlify Functions for the API?

Socket.io requires **persistent HTTP connections** for WebSocket handshaking and real-time events. Serverless functions terminate after each request (typically 10–30 seconds) and cannot hold a live connection. The API must run on a persistent server: Render, Railway, or Fly.io. All three have free tiers.

### Why HMAC-SHA256 JWTs (Not a Library)?

The token system in `middleware/auth.ts` uses Node’s built-in `crypto.createHmac` with `timingSafeEqual` for constant-time comparison. No third-party JWT library means no extra attack surface and no `jsonwebtoken` vulnerability surface. The implementation is straightforward and auditable.

### Why Drizzle ORM?

Drizzle generates TypeScript types directly from the schema — the DB schema *is* the types. It’s lightweight (no query builder overhead), has excellent TypeScript inference, and `drizzle-kit push` lets you apply schema changes without migration files during early development.

### Why Zustand for State?

React Context re-renders all consumers on every state update. Zustand uses a subscription model — components only re-render when the specific slice they read changes. For a game app with frequent score and card state changes, this matters.

### The Thunder Principle

*Thunder* is the 9th card type in the deck. It represents an unexpected moment of vulnerability or connection — not engineered, just made possible by the right conditions. The app’s design philosophy is built on this: good questions create the atmosphere; the connection is the couple’s own.
